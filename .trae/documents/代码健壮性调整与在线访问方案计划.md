# 代码健壮性优化与在线访问/导出功能实现计划

该计划旨在提升现有 Lead Generation 系统（SuperLink Data Engine）的稳定性，并增加远程访问及数据导出的核心功能。

## 一、 代码健壮性调整 (Robustness Enhancements)

### 1. 核心逻辑优化
- **环境配置修复**：修复 `requirements.txt` 的编码问题（转为 UTF-8），确保依赖库正常安装。
- **配置灵活性**：改进 `core/config.py`，将硬编码的代理设置为可选，并优先从环境变量读取。
- **AI 提取鲁棒性**：在 `core/processor.py` 中增强对 AI 返回 JSON 格式的容错处理（处理 Markdown 标记、空值清洗等）。
- **线程安全与并发**：优化 `AutomationManager` 的停止机制，确保任务中断时数据和进度的完整保存。

### 2. Web 界面 (Streamlit) 优化
- **输入校验**：增加对任务名称和搜索关键词的合法性校验。
- **监控体验**：改进自动化任务的监控逻辑，使用更优雅的状态更新方式代替简单的 `time.sleep` 轮询。
- **异常捕获**：在 UI 层增加更友好的错误提示（如 API 额度耗尽、网络超时等）。

## 二、 在线访问与导出功能 (Online Access & Export)

### 1. 文件导出功能实现
- **实时下载**：在任务完成后，增加 `st.download_button` 按钮，允许用户直接在浏览器中下载 CSV 文件。
- **历史记录管理**：在侧边栏或主界面增加“历史任务”面板，列出 `output/` 文件夹中的所有 CSV，方便用户随时查看和下载过往数据。

### 2. 在线访问方案
- **部署适配**：
    - 修改代码以支持 Streamlit Cloud 的部署方式（使用 `st.secrets` 代替本地 `.env`）。
    - 提供简单的身份验证方案（Streamlit 内部鉴权或简易登录页），防止 API 被非法调用。
- **访问建议**：
    - **快速分享**：使用 `localtunnel` 或 `ngrok` 进行内网穿透，实现即时在线访问。
    - **正式部署**：推荐使用 Streamlit Cloud (免费) 或私有 VPS (Docker 部署)。

## 三、 执行步骤
1. **第一阶段**：修复 `requirements.txt` 并优化 `core/` 下的逻辑代码。
2. **第二阶段**：修改 `app.py` 增加下载按钮和历史记录管理功能。
3. **第三阶段**：集成简单的访问控制并完善部署文档。
