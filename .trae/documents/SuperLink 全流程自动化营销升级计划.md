# SuperLink Data Engine 全流程自动化营销升级计划

根据最新的《SuperLink Data Engine升级方案》，我将分阶段为您实施此次深度升级。本次升级的核心目标是将系统从单一的“线索挖掘机”进化为“自动营销工厂”。

## 一、 升级内容深度分析
本次升级涵盖了营销全生命周期的四个关键环节：
1. **获取增强**：引入 LinkedIn 和工商数据，通过异步 IO 提升 10 倍采集效率。
2. **精准过滤**：通过 SMTP 握手和工商交叉验证，确保每一条线索的真实性，避免浪费 API 额度。
3. **自动触达**：集成个性化 HTML 邮件模板系统，模拟真人行为频率进行自动群发。
4. **效果闭环**：通过 IMAP 自动读取回复，利用 GLM-4 分析客户意向并生成可视化报表。

## 二、 技术路线与实施步骤

### 第一阶段：基础设施与验证引擎 (Verifier)
1. **数据库迁移**：从 `json` 文件存储迁移至 `SQLite`，建立 `verified_leads`、`email_records` 等核心表。
2. **实现 Verifier 模块**：创建 `core/verifier.py`，集成邮箱三重验证（格式+MX+SMTP）和官网抓取分析逻辑。
3. **依赖补全**：更新 `requirements.txt`，引入 `email-validator`, `dnspython`, `beautifulsoup4` 等库。

### 第二阶段：自动化触达引擎 (EmailSender)
1. **实现 EmailSender 模块**：创建 `core/email_sender.py`，集成 `yagmail` 和 `Jinja2` 模板引擎。
2. **模板系统**：创建 `templates/` 目录，设计标准的 B2B 开发信模板。
3. **安全机制**：实现限速发送和断点续传逻辑，保护发件箱信誉。

### 第三阶段：反馈分析与 AI 洞察 (FeedbackProcessor)
1. **实现 Feedback 模块**：创建 `core/feedback_processor.py`，通过 IMAP 获取回复。
2. **AI 意向分析**：调用 GLM-4 对邮件内容进行意向分类（高意向/咨询/拒收）。
3. **可视化升级**：在 Web 界面增加“营销看板”，展示回复率和意向统计。

### 第四阶段：全流程编排与隔离 (UI & Orchestration)
1. **调度器升级**：在 `automation.py` 中编排“搜索->验证->发送”的长流程任务。
2. **UI 汉化与集成**：在 `app.py` 中增加新的功能标签页，确保与现有功能完美融合。

## 三、 隔离性保证
- **数据隔离**：新的 SQLite 数据库将与旧的 CSV 导出并存。
- **开关逻辑**：所有自动化发送步骤均需用户手动确认开启，防止误发邮件。
