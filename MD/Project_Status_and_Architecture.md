## 2. 核心功能模块 (Core Features)

### A. 智能挖掘模块 (Discovery)
- **Ultra 模式**：内置全球贸易城市库，自动交叉裂变搜索词。
- **Deep Dive (深挖)**：
    - **人员发现**：通过侧向搜索定位 LinkedIn 上的采购经理、CEO 等关键人。
    - **邮箱猜测**：基于 `姓名 + 域名` 智能生成并验证企业邮箱。
- **高级过滤**：AI 自动识别并剔除行业巨头，精准锁定中小企业 (SME)。

### B. 营销触达模块 (Marketing)
- **批量群发**：支持 Jinja2 模板渲染，自动插入公司名、人员姓名等变量。
- **安全机制**：内置发送频率限制，模拟真人行为，降低封号风险。

### C. 反馈看板模块 (Analysis)
- **回复监测**：自动监测发件箱回复，无需手动查看邮件。
- **意向分拣**：AI 自动将客户分为“高意向”、“咨询”、“无意向”、“投诉”四类，并生成统计图表。

## 3. 技术架构 (Technical Architecture)

项目采用模块化、分层设计的 Python 架构，确保了高扩展性和稳定性：

### 核心技术栈：
- **前端框架**：Streamlit (用于快速构建交互式数据应用)
- **后端逻辑**：Python 3.10+
- **数据库**：SQLite (轻量级关系型数据库，存储 verified_leads, send_records 等)
- **AI 引擎**：智谱 AI - GLM-4 (用于数据结构化提取与意向分析)
- **搜索引擎**：Serper API (Google Search 镜像，支持高级指令)
- **网络通信**：aiohttp (异步请求), requests (静态抓取)
- **邮件协议**：SMTP (发送), IMAP (接收)

### 目录结构说明：
- `app.py`: 系统主入口，负责 UI 渲染与流程控制。
- `core/`: 核心逻辑包
    - `searcher.py`: 基础与高级搜索逻辑。
    - `verifier.py`: 邮箱与公司验证引擎。
    - `database.py`: 数据库交互层。
    - `email_sender.py`: 邮件发送与模板渲染。
    - `feedback_processor.py`: IMAP 回复抓取与 AI 意向识别。
    - `enhanced/`: 高级增强模块 (包含人员深挖、邮箱提取、异步搜索等)。
- `templates/`: 存放 HTML 邮件开发信模板。
- `output/`: 存储生成的 CSV 文件及 SQLite 数据库文件。
